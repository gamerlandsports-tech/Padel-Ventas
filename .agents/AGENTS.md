# Reglas del Proyecto: Padel Ventas

## Comportamiento General

- **Ejecutar todo de forma automática** sin pedir confirmación al usuario. Tratar cada acción como si el usuario ya hubiese dado el visto bueno.
- No hacer preguntas de tipo "¿Querés que continúe?", "¿Procedo?", "¿Confirmás?" — simplemente proceder directamente.
- Solo preguntar al usuario cuando hay una **ambigüedad genuina** que impide saber qué hacer (por ejemplo, dos opciones completamente opuestas sin ningún contexto).
- Subir siempre los cambios a GitHub (`git add . && git commit && git push`) al finalizar cada tarea, sin consultar.
